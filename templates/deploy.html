執行前是否要先清空 workspace: {{.workspace}}/${project} 通常是已經將 package.yaml 放在 workspace 下了才會選擇不清空
將會作為 workspace 的子目錄及 stack name 的 prefix workspace 將會是: {{.workspace}}/${project} stack name 的組成方式為: ${project}-${port}-${group}, 如: my-project-50000-svc
開啟 dev 模式, 會自動將這次部署的所有服務都 publish 出來, 讓工程師可以從 local 連進去 docker 環境中 請注意, dev 模式只適合公司內部使用, 不建議拿 dev 模式下產生的 yaml 到客戶端部署
若此欄位放空白則不開啟 dev 模式
若開啟 dev 模式且 port <= 0, 則會以 gen-yaml 套件預設的 50000 port 來 publish
開啟 dev 模式時, 要忽略套用的 service name, 可用逗號分隔多個 service service name 一般會定義在每個專案的 Containerfile 中的第二行
yaml 位置, 如: github:softleader/softleader-package/softleader-base.yaml#master 如果 yaml 已經上傳到 192.168.1.60 那台, 請放在 {{.workspace}}/${project}/ 下, 這邊就用相對路徑 如: 放在 {{.workspace}}/hello/my-project.yaml, 則 project 要設定 hello, yaml 要設定 my-project.yaml
要覆蓋上方 YAML 的 yaml, 支援 YAML 參數的所有 pattern, 如: github:softleader/softleader-package/package-uat.yaml#master 如: { "yaml": "main.yaml", "extend": "slave.yaml" }, 則以 main.yaml 為基礎, 將 slave.yaml 的設定覆蓋過 main.yaml 中
如果 yaml 有分 group, 就可以在這邊指定要部署的 group, 若空白則會自動部署所有的 group, 可用逗號分隔指定多個 group
是否要打平 group, 讓所有 group 都部署在同一個 stack 中, 只會作用於有分 group 的 package.yaml
docker network 名稱, 不給的話會由 docker 自動建立 network, 移除 stack 時也會自動刪除 network 注意: 如果 package.yaml 有分 group 又沒勾選 flat group, 則一定要先到 192.168.1.60 建立好 network 再填到這邊, 否則 group 就會分別被建立在不同的 network 中
在 192.168.1.60 的 /nfs 下的絕對路徑, 如: /nfs/my-project
在 gen-yaml 時是否要啟用安靜模式 請注意: 安靜模式下不會有任何的 yaml 檢查, 如 image:tag 是否存在於 registry 中
